# 商品画像を差し替えても表示が変わらないとき

## 想定される原因

### 1. ブラウザキャッシュ（最も多い）

**同じファイル名**（例: `1000.jpg`）で中身だけ差し替えると、URLが変わらないためブラウザが**古い画像をキャッシュから表示**し続けます。

**確認方法**
- シークレットウィンドウで開く、または **Ctrl+Shift+R（強制再読み込み）** で表示が変わるか試す。

**恒久対策（今回の対応）**
- `next.config.mjs` で `/images/products/*` に  
  `Cache-Control: public, max-age=0, must-revalidate` を付与しました。  
  差し替え後は、次回アクセス時にサーバーへ再取得し、新しい画像が表示されやすくなります。  
  **反映にはサーバーでビルドし直しが必要です。**

---

### 2. 画像を置いている場所が違う

Docker では **ホストの次のパス** がコンテナの `public/images/products` にマウントされています。

- 正しい場所（例）:  
  `/mnt/nvme01/project/108teaworks-vibe/next-app/public/images/products/<商品スラッグ>/`
- ここに置いたファイルだけが、コンテナ内の `/app/public/images/products/<スラッグ>/` として参照されます。

**確認**
- デザイナーが更新しているのは、**このサーバー上のパス**か。
- 別マシンで差し替えているだけなら、そのファイルをこのパスにアップロード（FTP/SCP 等）する必要があります。

---

### 3. ファイル名の並び順

商品の「メイン画像」は、そのフォルダ内のファイルを**名前でソートしたときの先頭**が使われます。

- 例: `1000.jpg` と `1000.png` が両方あると、辞書順で先になる方が表示されます。
- 差し替え時は **古い方のファイルを削除**するか、新しいファイルの名前を辞書順で先になるようにしてください（例: `00_main.png`）。

---

### 4. 再起動で変わる理由

- 再起動すると、**新しいタブや再読み込み**で見ることが多く、そのタイミングでキャッシュをバイパスしている可能性があります。
- 上記の **Cache-Control 設定** を入れてビルドし直せば、再起動なしでも「次回アクセス時に新しい画像が出る」ようにしやすくなります。

---

## 運用の目安

1. 画像は **マウント元の正しいパス** に配置する。
2. 同じ名前で差し替える場合は、**古いファイルを消してから**新しいファイルを置く。
3. 表示確認時は **強制再読み込み（Ctrl+Shift+R）** またはシークレットで確認する。
4. `next.config.mjs` の変更後は、サーバーで **ビルドし直してから** 反映を確認する。
